## Critical Anti-Patterns (each discovered by an actual failure/finding this session)

| Pattern | Description | Severity | Prevention |
|---------|-------------|----------|------------|
| API key in the config body | The BYOK key must NOT go in the `config` JSON — that object is persisted as the job's `run_config` (`store.create`), so it would leak to disk/logs. | blocking (for the backend) | Key rides as a transport-only `x-anthropic-key` header only; held in memory for the job, never persisted. |
| Mutating `os.environ` for per-request keys | Core `anthropic.Anthropic()` reads `ANTHROPIC_API_KEY` from env in **4 places** (`src/ddharmon/llm/batch.py` ×3, `llm/anthropic_client.py:64`). Setting env per-request races + leaks keys across concurrent jobs. | blocking (for the backend) | Thread the key down to each client constructor as `api_key=...`; never touch process env. |
| Tailwind arbitrary `clamp()` minus operator | `clamp(1rem,16vh-117px,3.5rem)` is invalid CSS — `calc`-style math needs spaces around `-`. | advisory (fixed) | Use underscores in the arbitrary value: `lg:py-[clamp(1rem,16vh_-_117px,3.5rem)]` → Tailwind expands `_`→space. |
| Force-fit-to-viewport layout | Pinning a multi-section page to 100vh / `flex-1` fill is fragile across zoom + browser chrome and needs magic pixel offsets. | advisory | Use fluid `clamp()` spacing keyed to vh; floors = the compact case, capped maxima. Let it scroll on genuinely tiny windows. |
| Playwright module resolution | `import "playwright"` from a scratchpad script fails — ESM resolves from the script's dir, and Playwright is only in `frontend/node_modules`. | advisory | Put screenshot scripts inside `frontend/` (or run from there) so the bare import resolves. |

**Demo (AoU/CLSA/UKBB, 200 fields each → 350 records):** built by the real production pipeline (paid batch,
~$3-4 incl. drift). Works both ways:
- **Backend replay** — `start_demo` paces the precomputed snapshot through the pipeline phases via the existing
  JobStore/SSE (stable per-combo job id `demo-aou_clsa_ukbb`, `config.demo=true`); no LLM/key.
- **Static (Netlify) replay** — `build_demos.py --fixtures-only` emits `static-data/result-demo-*.json` (bundled
  in the `VITE_STATIC` build) + merges the demo into `static-data/jobs.json`; `useHarmonizeStream` does a
  **client-side paced replay** so the backend-less site feels live. `startDemo` returns the deterministic id.
- Frontend: single **"Load demo"** (no cohort checkboxes); Runs page **"Demo" badge**.

**Reproducibility (byte-identical by construction):** `run_pipeline` freezes/reloads the clustering **substrate**
(skips UMAP on replay); `_batch_stage` uses **`resume_and_wait`** (reuses cached responses). Proven: ship + 2
replays **byte-identical**. A **fresh** run drifts (measured: 350→320 records, **298/350 recur, 260/298 same
verdict** ≈ 85%/87%) because UMAP/HDBSCAN + LLM aren't bitwise-reproducible — largely absorbed by split-aware
assignment. A **"Reproducibility" popover** on every run's header states this (with the measured numbers).

**Run-options cleanup:** removed **min-cluster-size** from the form — the engine now **auto-scales** it from
corpus size (`_auto_min_cluster_size`, `max(3, min(15, round(N/150)))`; ~4 for the 600-field demo, 15 for large
cohorts; explicit still wins). Dropped **Sync** from the run-mode picker (Batch + Preview only; backend still
accepts sync via API). Requires core `resume_and_wait(api_key=)` (research repo @ `af2c010`).
</demo_this_session>
